# Интеграция с МойСклад

Руководство по подключению бонусной системы Gupil к МойСклад для синхронизации бонусов между онлайн- и офлайн-продажами.

> **Direct API** — рекомендуемый способ. Gupil напрямую обращается к API МойСклад и принимает webhook'и о бонусных операциях, обеспечивая двустороннюю синхронизацию баланса.

## Что вам понадобится

- Доступ к админ-панели Gupil и созданный проект
- Аккаунт МойСклад с подключённой программой лояльности
- API-токен МойСклад (Настройки → Токены доступа)
- ID программы лояльности (Bonus Program ID)

## Шаг 1: Получите данные в МойСклад

1. Войдите в МойСклад
2. **Настройки → Токены доступа** — создайте токен и скопируйте его
3. **Account ID** — идентификатор вашего аккаунта (обычно email учётной записи)
4. **Bonus Program ID** — UUID программы лояльности из адресной строки соответствующего раздела

## Шаг 2: Создайте интеграцию в Gupil

1. В Gupil откройте **Проекты** → выберите проект
2. Перейдите на вкладку **Интеграция** → **МойСклад**
3. Заполните форму:
   - **API-токен** (будет сохранён в зашифрованном виде)
   - **Account ID**
   - **Bonus Program ID** (формат UUID)
   - **Направление синхронизации**: двусторонняя, только из МойСклад или только в МойСклад
   - **Автосинхронизация** — включить/выключить
4. Нажмите **Сохранить**, затем **Проверить подключение** — должно появиться уведомление об успехе

## Шаг 3: Настройте webhook в МойСклад

1. В Gupil скопируйте **Webhook URL** интеграции
   - Формат: `https://gupil.ru/api/webhook/moysklad-direct/ВАШ_PROJECT_ID`
2. В МойСклад настройте отправку webhook о бонусных операциях на этот URL

> Для безопасности Gupil перезапрашивает каждую транзакцию из API МойСклад — поддельные данные webhook не приведут к начислению.

## Что умеет интеграция

- ✅ **Двусторонняя синхронизация** — бонусы из офлайн-продаж в МойСклад попадают в Gupil и наоборот
- ✅ **Автосвязывание клиентов** — пользователи связываются по номеру телефона
- ✅ **Учёт срока действия** — начисленные бонусы получают `expiresAt` из настроек проекта
- ✅ **Сверка балансов** — проверка соответствия балансов между системами
- ✅ **Журнал операций** — все синхронизации логируются с request/response
- ✅ **Ручная синхронизация** — кнопка «Синхронизировать» на странице интеграции
- ✅ **Безопасное хранение токена** — шифрование AES-256-GCM

## Проверка работы

1. **Подключение**: нажмите **Проверить подключение** — должно вернуться «Подключение успешно»
2. **Синхронизация**: нажмите **Синхронизировать** и проверьте счётчики «Успешно / Ошибок» в уведомлении
3. **Логи**: откройте таблицу логов на странице интеграции и раскройте записи для просмотра деталей
4. **Баланс**: оформите тестовую офлайн-продажу с бонусами в МойСклад и убедитесь, что баланс клиента обновился в Gupil

## Решение проблем

### «Ошибка подключения» при проверке
- Проверьте корректность API-токена и Account ID
- Убедитесь, что токен не отозван в МойСклад
- Проверьте, что Bonus Program ID указан в формате UUID

### Бонусы не синхронизируются
- Проверьте **направление синхронизации** (двусторонняя/одностороннее)
- Убедитесь, что webhook в МойСклад настроен на правильный URL
- Посмотрите журнал операций интеграции — там видна причина ошибки

### Клиент не связывается
- Связывание идёт по номеру телефона — убедитесь, что телефон совпадает в обеих системах в одинаковом формате

## Требования к окружению

Для работы шифрования токенов на сервере должна быть задана переменная окружения `MOYSKLAD_ENCRYPTION_KEY` (надёжный ключ от 32 символов). Это настраивается администратором при деплое.
