# Партнёрская программа (B2B)

Многоуровневая партнёрская модель поверх реферальной системы: производитель → директора → менеджеры → тренеры → клиенты, с автоматическим начислением комиссии по цепочке вверх.

> Режим включается **отдельно для каждого проекта** и по умолчанию выключен. Проекты без него работают как обычная c2c-реферальная программа без изменений.

## Когда использовать

Подходит для сетевых и партнёрских продаж, где есть несколько уровней представителей, и каждый уровень получает свой процент от покупок привлечённых клиентов. Например, фитнес-сеть: тренер приводит клиента, а менеджер и директор тренера тоже получают долю.

## Роли партнёров

- **CLIENT** (по умолчанию) — обычный клиент, не может приглашать и не получает комиссию.
- **TRAINER** (тренер) — выдаёт реферальную ссылку, получает комиссию первого уровня.
- **MANAGER** (менеджер) — получает комиссию со своих тренеров (второй уровень).
- **DIRECTOR** (директор) — получает комиссию со всей команды (третий уровень).

## Как работает начисление

1. Клиент регистрируется по ссылке партнёра.
2. При покупке клиента комиссия начисляется вверх по цепочке приглашающих — каждому уровню свой процент.
3. Глубина выплат ограничена параметром **maxPayoutDepth** (по умолчанию 3 уровня).
4. Проценты берутся из персонального плана партнёра, либо из плана комиссий проекта по умолчанию.

**Пример:** покупка клиента на 5000 ₽ → тренер 7% (350 ₽) + менеджер 2% (100 ₽) + директор 1% (50 ₽).

> Клиенты (CLIENT) в цепочке пропускаются — даже если клиент указан как пригласивший, комиссия идёт следующему партнёру выше.

## Включение для проекта

1. Откройте **Проекты** → проект → **Настройки**.
2. Включите переключатель **B2B-режим** (партнёрские роли).
3. На странице реферальной программы задайте план комиссий по умолчанию и глубину выплат.
4. Назначьте роли пользователям в разделе **Пользователи** (колонка роли, селектор в карточке клиента). Можно массово «Назначить всем тренерам».

## Управление и просмотр

- **Пользователи**: фильтр по роли, назначение роли и персонального плана.
- **Реферальная программа → Иерархия** (`/referral/hierarchy`): дерево партнёров с поиском, выбором периода, статистикой по команде и CSV-экспортом.
- **Права доступа**: партнёр видит свою статистику и статистику нижестоящих в своей ветке (до maxPayoutDepth); дополнительно доступ можно выдать вручную.

## Кабинет партнёра в Telegram

Партнёрам доступен кабинет прямо в боте (готовый шаблон workflow «🏢 B2B Кабинет партнёра» с меню по роли):

- **Моя команда** — список нижестоящих партнёров и клиентов.
- **Статистика** — по выбранному участнику команды.
- **Выплаты** — начисленные комиссии.
- **Моя ссылка** — реферальная ссылка партнёра.
- **Сводка по организации** — для директоров.

## Уведомления

- При новой регистрации в команде уведомление получает всё дерево предков, с текстом по уровню (новый клиент / у вашего тренера новый клиент / новая регистрация в организации).
- При начислении реферальной комиссии партнёр видит сумму, имя клиента-источника и уровень.
- Отключить уведомления о реферальных событиях можно в настройках пользователя.

## Откат

Режим можно выключить в настройках проекта в любой момент — поведение вернётся к обычной реферальной программе, а роли в базе сохранятся.
