# Бонусная система

Руководство по управлению программой лояльности.

## Принципы работы

1. **Регистрация** — начисление приветственных бонусов.
2. **Покупка** — начисление процента от суммы.
3. **Оплата** — списание бонусов.
4. **Срок действия** — сгорание неиспользованных бонусов.

## Настройка

Раздел: `/settings`

### Параметры
- **Процент начисления**: Доля от суммы заказа в бонусах.
- **Срок действия**: Период в днях до сгорания (если не задан — бонусы бессрочные).
- **Приветственный бонус**: Начисление за регистрацию.

### Режим расчёта процента (bonusMode)

- **SIMPLE (простой)**: единый процент для всех клиентов, берётся из параметра «Процент начисления». Быстрая настройка за минуту — подходит для большинства программ.
- **LEVELS (уровни)**: процент зависит от суммы покупок клиента и определяется его уровнем лояльности (см. ниже).

При переключении режима существующие бонусы клиентов не меняются — меняется только логика начисления новых. Уровни при переходе на SIMPLE деактивируются, но не удаляются.

### Режимы работы (BonusBehavior)

- **EARN_ONLY**: Только накопление.
- **SPEND_ONLY**: Только списание (актуально при закрытии программы).
- **SPEND_AND_EARN**: Одновременное накопление и списание (базовый режим).

*При режиме SPEND_AND_EARN, если клиент частично оплатил заказ бонусами, новые бонусы начисляются только на остаток, оплаченный деньгами. При SPEND_ONLY с использованием бонусов начисление не происходит.*

## Уровни лояльности

Раздел: `/bonus-levels`

Позволяет менять процент начисления в зависимости от общей суммы покупок клиента.

| Уровень | Мин. сумма | % бонусов | Макс. оплата |
|---------|------------|-----------|--------------|
| Базовый | 0₽ | 5% | 10% |
| Серебряный | 50,000₽ | 10% | 25% |
| Золотой | 150,000₽ | 15% | 40% |

Система автоматически пересчитывает уровень при каждой покупке. Уровни работают, когда выбран режим расчёта **LEVELS**.

## Реферальная программа

Раздел: `/referral`

Многоуровневая система вознаграждений за приглашение пользователей.

**Параметры:**
- Проценты для уровней (1-3 уровня).
- Приветственный бонус рефералу.
- Срок действия реферальной связи (cookie).

### Персональные планы комиссий (блогеры/инфлюенсеры)

Для отдельных партнёров можно назначить персональный план реферальных процентов, отличный от общей программы. План фиксируется снимком при регистрации реферала по ссылке, что защищает от изменений задним числом. Раздел: вкладка «Планы %» на странице реферальной программы.

### B2B-иерархия партнёров

Опциональный режим (включается per project) с ролями **CLIENT / TRAINER / MANAGER / DIRECTOR** и начислением комиссии по цепочке вверх (например, тренер 7% + менеджер 2% + директор 1%). Подробнее — в разделе [Партнёрская программа (B2B)](/partner-program).

## Срок действия и сгорание бонусов

- Срок задаётся параметром «Срок действия» в днях; при начислении бонус получает дату `expiresAt`.
- Истёкшие бонусы исключаются из доступного баланса.
- Списание идёт по принципу **FIFO** — сначала тратятся бонусы с ближайшим сроком сгорания.

## Управление бонусами

Раздел: `/users`

### Операции администратора
- **Начисление**: Ручное добавление бонусов (подарок, компенсация).
- **Списание**: Ручное уменьшение баланса.
- **Корректировка**: Установка точного значения баланса.

### История транзакций
Типы операций: `EARNED` (покупка), `SPEND` (трата), `EXPIRED` (сгорание), `WELCOME` (регистрация), `REFERRAL` (реферал), `BIRTHDAY` (бонус ко дню рождения), `ADMIN` (ручная).
