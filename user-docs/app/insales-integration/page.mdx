# Интеграция с InSales

Руководство по подключению бонусной системы Gupil к интернет-магазину на платформе InSales.

> Интеграция работает по двум каналам: **webhook'и** InSales (начисление бонусов за заказы и регистрация клиентов) и **JavaScript-виджет** на витрине (баланс и оплата бонусами).

## Что вам понадобится

- Доступ к админ-панели Gupil и созданный проект
- Магазин на InSales и доступ к его настройкам
- API-ключ и пароль InSales (Настройки → API)

## Шаг 1: Получите API-доступ в InSales

1. Войдите в админ-панель InSales
2. Перейдите в **Настройки → API**
3. Создайте новый API-ключ
4. Скопируйте **API Key** и **API Password**
5. Запомните домен магазина в формате `yourshop.myinsales.ru`

## Шаг 2: Активируйте интеграцию в Gupil

1. В Gupil откройте **Проекты** → выберите проект
2. Перейдите на вкладку **Интеграция** → **InSales**
3. Заполните форму:
   - **Учётные данные API**: API Key, API Password, Домен магазина
   - **Использовать настройки проекта** — если включено, процент начисления и лимит оплаты бонусами берутся из общих настроек проекта; если выключено — задаются отдельно
   - **Процент начисления бонусов (%)** и **Максимум оплаты бонусами (%)** — при отключённом наследовании
   - **Виджет** и **Бейджи на товарах** — включение виджета и плашек «начислим бонусы»
4. Нажмите **Активировать**

## Шаг 3: Настройте webhook'и в InSales

1. В Gupil в блоке **Настройка Webhooks в InSales** скопируйте **Webhook URL**
   - Формат: `https://gupil.ru/api/insales/webhook/ВАШ_PROJECT_ID`
2. В InSales перейдите в **Настройки → Webhooks**
3. Добавьте webhook'и на события:
   - **Создание заказа** (`orders/create`)
   - **Обновление заказа** (`orders/update`)
   - **Создание клиента** (`clients/create`)
4. Укажите скопированный URL для каждого события

## Шаг 4: Встройте виджет на сайт

1. В Gupil в блоке **Код виджета для сайта** нажмите **Копировать**
2. В InSales перейдите в **Дизайн → Редактор тем**
3. Откройте `layout.liquid` (или `theme.liquid`)
4. Вставьте код виджета **перед закрывающим тегом** `</body>`
5. Сохраните изменения

```html
<!-- InSales Bonus Widget -->
<script
  src="https://gupil.ru/insales-widget-loader.js"
  data-project-id="ВАШ_PROJECT_ID"
></script>
```

## Что умеет интеграция

- ✅ **Начисление бонусов за заказы** — при оплаченном заказе клиенту начисляются бонусы согласно проценту проекта или интеграции
- ✅ **Регистрация клиентов** — новые клиенты из InSales создаются в Gupil автоматически, с приветственными бонусами (если настроены)
- ✅ **Оплата бонусами** — виджет показывает доступный баланс и позволяет применить бонусы в корзине
- ✅ **Бейджи на товарах** — плашки «начислим до N бонусов» на карточках товаров
- ✅ **Логика BonusBehavior** — учитывается режим проекта (начисление/списание) при расчёте бонусов
- ✅ **Защита от дублей** — повторные `orders/update` не приводят к повторному начислению

## Проверка работы

1. **Виджет**: откройте витрину магазина — в корзине должен отображаться блок бонусов
2. **Webhook**: оформите тестовый заказ и проверьте в Gupil раздел логов InSales (статус 200)
3. **Клиент**: убедитесь, что клиент появился в разделе **Пользователи** проекта
4. **Начисление**: проверьте, что в транзакциях клиента есть запись о начислении бонусов

## Решение проблем

### Webhook не приходит
- Проверьте, что URL скопирован полностью и содержит правильный `projectId`
- Убедитесь, что интеграция в Gupil **активна**
- Проверьте, что в InSales webhook'и добавлены на нужные события

### Бонусы не начисляются
- Проверьте процент начисления в настройках интеграции/проекта
- Убедитесь, что заказ оплачен (`payment_status: paid`)
- Посмотрите логи InSales в Gupil — там виден разобранный payload и ошибка

### Виджет не отображается
- Проверьте, что код вставлен перед `</body>` и сайт сохранён
- Убедитесь, что `data-project-id` указан верно
- Откройте консоль браузера (F12) и проверьте загрузку скрипта виджета
